# Git / GitHub — рабочая инструкция для проекта Travel Radar LK

Этот файл нужен, чтобы:
- не вспоминать каждый раз, как настроен GitHub
- не искать команды по чатам
- работать с Git спокойно и стабильно

====================================================================

ОБЩЕЕ

- GitHub больше НЕ принимает пароль через веб
- Аутентификация паролем отключена навсегда
- Рабочий и правильный способ — SSH
- Токены (PAT) НЕ используются
- Проблемы с 90 днями отсутствуют (SSH не истекает)

====================================================================

ТЕКУЩАЯ КОНФИГУРАЦИЯ

- Операционная система: Windows
- Терминал: PowerShell / Antigravity
- Репозиторий: travelradar-lk
- Аутентификация: SSH
- Используемый ключ:
  C:\Users\PC\.ssh\id_travelradar

Git настроен использовать этот ключ глобально:

git config --global core.sshCommand "ssh -i C:/Users/PC/.ssh/id_travelradar"

====================================================================

ПРОВЕРКА SSH (если есть сомнения)

Команда:

ssh -T git@github.com

Ожидаемый ответ:

Hi leonid-cordon! You've successfully authenticated, but GitHub does not provide shell access.

Если это сообщение есть — SSH работает.

====================================================================

ОБЫЧНЫЙ РАБОЧИЙ ЦИКЛ (КАЖДЫЙ ДЕНЬ)

git status
git add .
git commit -m "Краткое описание изменений"
git push

- логин не спрашивается
- пароль не спрашивается
- подтверждений нет

====================================================================

ПРОВЕРКА REMOTE (ИНАЧЕ PUSH НЕ ПОЙДЁТ)

git remote -v

Должно быть:

git@github.com:leonid-cordon/travelradar-lk.git

Если вдруг https — заменить:

git remote set-url origin git@github.com:leonid-cordon/travelradar-lk.git

====================================================================

ЕСЛИ git push ВДРУГ ПЕРЕСТАЛ РАБОТАТЬ

1. Проверить SSH:

ssh -T git@github.com

Если приветствие есть — проблема НЕ в ключе.

2. Проверить remote:

git remote -v

3. НИЧЕГО не менять с паролями и токенами.

====================================================================

ПРО ТОКЕНЫ И 90 ДНЕЙ

- Personal Access Token (PAT) здесь НЕ используется
- Токены имеют срок действия (90 дней и т.п.)
- SSH-ключ НЕ имеет срока действия
- Поэтому проект работает ТОЛЬКО через SSH

====================================================================

ЕСЛИ СМЕНИТСЯ КОМПЬЮТЕР

Нужно:
1. Создать новый SSH-ключ
2. Добавить его в GitHub → Settings → SSH and GPG keys
3. Настроить Git на новый ключ

Старые ключи можно оставить или удалить — по желанию.

====================================================================

ЧЕГО НЕ ДЕЛАТЬ

- НЕ использовать пароль GitHub
- НЕ возвращаться к HTTPS
- НЕ хранить токены в файлах
- НЕ удалять id_travelradar, если всё работает

====================================================================

ИТОГ

- SSH настроен один раз
- GitHub работает стабильно
- Авторизация больше не проблема
- Можно спокойно заниматься проектом
